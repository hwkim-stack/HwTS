"미미" 루프의 **기획** 단계다. 목표: AdSense 수익을 늘릴 다음 유틸리티 도구 **1개**를 정한다.

1. `mimi/state.json` 의 기존 도구 목록과 `mimi/metrics.csv`(있다면)를 읽는다.
2. `node scripts/report.mjs` 로 어떤 종류가 잘 되는지 확인한다.
3. 새 도구 후보 **3개**를 다음 기준으로 제안한다:
   - 한국 사용자가 실제로 많이 검색하는가(롱테일 키워드 우선)
   - 100% 클라이언트 JS 만으로 구현 가능한가(백엔드/유료 API 불필요)
   - 기존 도구와 겹치지 않고 상호 링크로 시너지가 나는가
4. 후보별로 (slug · 제목 · 타깃 키워드 · 한 줄 이유)를 표로 제시하고 **1개를 추천**한다.
5. 사용자가 고르면 `mimi/state.json` 의 `plan` 을 `{ "slug": ..., "title": ..., "keywords": [...], "why": ..., "step": "build" }` 로 채우고 `currentStep` 을 `"build"` 로 바꾼다. `log` 에도 한 줄 남긴다.
6. 마지막에 "다음: /mimi-build" 를 안내한다.

사용자가 인자로 아이디어를 줬다면 우선 반영한다: $ARGUMENTS

참고 후보 풀(중복 피해서 고르기): 단위변환, 날짜계산기(D-day), 나이계산기, BMI계산기, 할인가계산기, 부가세계산기, 환율(고정표)·시간대 변환, JSON 포맷터, 색상 HEX↔RGB, 로렘입숨 생성, QR(텍스트→링크), 타이머/스톱워치, 랜덤 추첨, 줄정렬/중복제거.
